# ADR-003: Sandbox Execution Model

* **Status:** Accepted
* **Date:** 2025-01-01

## Context
Calculator expressions previously executed inline via `asteval` inside the web application process. This exposed the service to malicious expressions (e.g., accessing builtins) and unbounded resource consumption.

## Decision
Execute expressions inside a subprocess created with `multiprocessing.Process` and communicate via `Pipe`. Within the subprocess we:
- Use `asteval.Interpreter` with a curated whitelist of math helpers from `calculator_logic`.
- Clear the symtable before each execution and repopulate with allowed functions and sanitized context variables.
- Apply `resource.setrlimit` to cap address space usage and rely on parent process to enforce execution timeouts (kill after `max_runtime_seconds`).
- Enforce AST validation and complexity budgets prior to sandbox invocation.

## Consequences
* Host process remains resilient even if evaluation crashes or exceeds memory limits.
* Guardrails are centralized in `calculator_core`, making policy updates reusable across future services.
* Slight overhead from process creation, acceptable for current throughput targets; future phases can pre-warm worker pools if needed.
